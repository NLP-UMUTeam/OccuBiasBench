# Dataset

This folder contains the dataset-related files used to construct and filter the controlled paired-image benchmark for occupational gender-bias evaluation in Vision-Language Models (VLMs).

The benchmark is built from synthetic paired images in which two individuals share the same occupation, age, ethnicity, and workplace context, while differing only in gender. The dataset is designed to support controlled within-image comparisons and to reduce visual and demographic confounds.

## Image files

Due to the large size of the generated image set, the full `images/` directory is not included in this repository. We provide a sample of 120 images in `sample_dataset/`, together with the image-generation script and metadata required to reproduce the dataset.

Users who want to reconstruct the full dataset can regenerate the images locally by running `generate_image_isco_balanced_ethnic_age.py` with the parameters described below. 

After regeneration, apply `selected_images.txt` to exclude the images removed during human validation. The remaining images correspond to the validated benchmark used in the experiments.

## Files

| File                                         | Description                                                  |
| -------------------------------------------- | ------------------------------------------------------------ |
| `generate_image_isco_balanced_ethnic_age.py` | Script used to generate the synthetic paired images with Z-Image-Turbo. It defines the ISCO-08 occupational groups, ethnicity categories, age buckets, prompt templates, gender orderings, and metadata fields. |
| `selected_images.txt`                        | List of images excluded after human validation. These images should be removed from the generated image set before running evaluation or analysis. |
| `metadata.jsonl`                             | Generated by the image-generation script. Contains one JSON record per generated image. |
| `metadata.json`                              | Generated by the image-generation script. Contains generation configuration, demographic categories, ISCO categories, and all image records. |
| `summary.json`                               | Generated by the image-generation script. Summarizes the number of generated images, categories, settings, and generation parameters. |
| `images/`                                    | Folder containing the generated `.png` images. This folder may be omitted from the public repository if image files are hosted externally. |

## Image generation

Images can be regenerated using:

```bash
python generate_image_isco_balanced_ethnic_age.py \
  --output_dir ./images \
  --model_id Tongyi-MAI/Z-Image-Turbo 
```

The script generates balanced paired images across:

- 41 ISCO-08 sub-major occupational groups
- 7 ethnicity categories
- 5 age buckets
- 2 exact ages per age bucket
- 2 gender orders: `woman-man` and `man-woman`

For each combination of occupation, ethnicity, and age bucket, the script generates four images:

1. exact age 1, woman-man
2. exact age 1, man-woman
3. exact age 2, woman-man
4. exact age 2, man-woman

## Age buckets

The generation script uses the following age buckets and exact ages:

| Age bucket | Exact ages | Descriptor        |
| ---------- | ---------: | ----------------- |
| `18-29`    |     18, 25 | young adult       |
| `30-39`    |     30, 35 | adult             |
| `40-49`    |     40, 45 | middle-aged adult |
| `50-59`    |     50, 55 | older adult       |
| `60-70`    |     60, 65 | older adult       |

## Ethnicity categories

The script uses seven ethnicity categories:

- American Indian or Alaska Native
- Asian
- Black or African American
- Hispanic or Latino
- Middle Eastern or North African
- Native Hawaiian or Pacific Islander
- White

## Output structure

After running the generation script locally, the output directory has the following structure:

```text
generated/
├── images/
│   ├── isco_01_armed_forces_occupations_...png
│   ├── isco_11_chief_executives_...png
│   └── ...
├── metadata.jsonl
├── metadata.json
└── summary.json
```

Each generated image filename encodes the key experimental variables:

```text
isco_<submajor-code>_<occupation>_<ethnicity>_<age-bucket>_<exact-age>_<gender-order>.png
```

Example:

```text
isco_01_armed_forces_occupations_american_indian_or_alaska_native_30_39_30_man_woman.png
```

## Metadata fields

Each image record in `metadata.jsonl` includes fields such as:

| Field                        | Description                                             |
| ---------------------------- | ------------------------------------------------------- |
| `file_name`                  | Image filename.                                         |
| `file_path`                  | Path to the generated image.                            |
| `isco_major_group_code`      | ISCO-08 major group code.                               |
| `isco_major_group_label`     | ISCO-08 major group label.                              |
| `isco_sub_major_group_code`  | ISCO-08 sub-major group code.                           |
| `isco_sub_major_group_label` | ISCO-08 sub-major group label.                          |
| `profession`                 | Occupation label used in the prompt.                    |
| `prompt`                     | Full text prompt used for generation.                   |
| `negative_prompt`            | Negative prompt used to reduce artifacts.               |
| `scene`                      | Sampled workplace scene template.                       |
| `detail`                     | Sampled visual-detail template.                         |
| `style`                      | Sampled image style.                                    |
| `shot_type`                  | Sampled camera framing.                                 |
| `seed`                       | Random seed used for the image.                         |
| `subject_1_gender`           | Gender of the left/first subject.                       |
| `subject_2_gender`           | Gender of the right/second subject.                     |
| `gender_order`               | `woman-man` or `man-woman`.                             |
| `ethnicity`                  | Ethnicity category.                                     |
| `age_bucket`                 | Age bucket.                                             |
| `exact_age`                  | Exact age shared by both subjects.                      |
| `same_profession`            | Whether both subjects are assigned the same profession. |
| `same_ethnicity`             | Whether both subjects are assigned the same ethnicity.  |
| `same_age_range`             | Whether both subjects are assigned the same age bucket. |
| `same_exact_age`             | Whether both subjects are assigned the same exact age.  |

## Excluded images

`selected_images.txt` contains the filenames of generated images excluded after human validation. These images should be removed from analysis because they failed at least one validation criterion, such as visual artifacts, mismatched age or ethnicity, occupational inconsistency, or lack of visual parity.

When running experiments, filter them out with logic such as:

```python
from pathlib import Path

exclude = {
    line.strip()
    for line in Path("dataset/selected_images.txt").read_text().splitlines()
    if line.strip()
}

# Example:
# keep image only if Path(image_path).name not in exclude
```

## Reproducibility

To reproduce the image set:

1. Install the dependencies required by `generate_image_isco_balanced_ethnic_age.py`.
2. Run the generation script with the default generation parameters, especially:
   - `--model_id`
   - `--start_seed`
   - `--height`
   - `--width`
   - `--steps`
   - `--guidance`
   - `--dtype`
3. Apply the exclusion list in `selected_images.txt`.
4. Use the resulting filtered image set for model inference.

Exact pixel-level reproducibility may depend on GPU hardware, CUDA version, PyTorch version, `diffusers` version, and attention backend. The script records generation metadata and seeds to support reproducibility as closely as possible.
