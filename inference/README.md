# Inference

This folder contains the scripts used to run zero-shot VLM inference on the generated paired-image benchmark.

## Files

| File           | Description                                                  |
| -------------- | ------------------------------------------------------------ |
| `VLMBase.py`   | Generic VLM wrapper for Gemma-3, Qwen3-VL, LLaVA, and InternVL models. Model aliases are mapped to Hugging Face model IDs. |
| `zero_shot.py` | Runs zero-shot inference over a metadata JSONL file and writes predictions to JSONL output files. |

## Hugging Face model aliases

The wrapper uses short aliases in the command line and maps them to Hugging Face model IDs:

## Expected input

The inference script expects a metadata JSONL file generated by the dataset script:

```text
../dataset/generated/metadata.jsonl
```

Each record should contain at least:

- `file_path`
- `subject_1_gender`
- `subject_2_gender`
- `subject_1_age`
- `subject_2_age`
- `gender_order`
- `ethnicity`

The image path is resolved using `--image_base_dir`, which defaults to:

```text
../dataset/generated/images
```

## Example usage

From inside this folder:

```bash
python zero-shot.py \
  --input_jsonl ../dataset/metadata.jsonl \
  --image_base_dir ../dataset \
  --prompt_file_1 ../prompts/indirect_prompt.txt \
  --prompt_file_2 ../prompts/explicit_prompt.txt \
  --output_jsonl_prompt1 ../results/indirect/predictions_indirect_prompt_qwen3_vl_4b.jsonl \
  --output_jsonl_prompt2 ../results/explicit/predictions_explicit_prompt_qwen3_vl_4b.jsonl \
  --model_name Qwen3-VL-4B-Instruct \
  --quantization_config none \
  --max_new_tokens 1024
```


## Structured reasoning inference

The same `zero_shot.py` script can also be used to run the structured reasoning prompts. In this case, use the reasoning prompt files and write the outputs to the corresponding `results/reasoning/` folders.

Example for Qwen3-VL-4B:

```bash
python zero-shot.py \
  --input_jsonl ../dataset/metadata.jsonl \
  --image_base_dir ../dataset \
  --prompt_file_1 ../prompts/reasoning_spatial_prompt.txt \
  --prompt_file_2 ../prompts/reasoning_gender_prompt.txt \
  --output_jsonl_prompt1 ../results/reasoning/spatial/predictions_reasoning_spatial_qwen3_vl_4b_analysis.jsonl \
  --output_jsonl_prompt2 ../results/reasoning/gender/predictions_reasoning_gender_qwen3_vl_4b_analysis.jsonl \
  --model_name Qwen3-VL-4B-Instruct \
  --quantization_config none \
  --max_new_tokens 4096
```

Recommended output paths for structured reasoning are:

```text
../results/reasoning/spatial/
../results/reasoning/gender/
```

Reasoning outputs are longer than the final comparison outputs, so `--max_new_tokens` should usually be increased, for example to `4096` or higher depending on the model.

For reasoning runs, the model is expected to return a structured JSON response containing:

```text
scene_description
individual_evaluation
comparison
```

The `individual_evaluation` field contains the 1--5 attribute scores for both people. The `comparison` field contains either:

```text
left | right | similar
```

for the spatial reasoning prompt, or:

```text
man | woman | similar
```

for the gender-explicit reasoning prompt.

These outputs are later used by the evaluation scripts to compute Reasoning Attribute Score Bias (RAS).

## Output format

The script writes one JSON object per image. Each output record includes:

- original metadata fields needed for evaluation,
- `resolved_image_path`,
- `llm_prediction`,
- `llm_reasoning`.

For the indirect prompt, `llm_prediction` should be normalized later to:

```text
left | right | cannot
```

For the explicit prompt, `llm_prediction` should be normalized later to:

```text
man | woman | cannot
```

For structured reasoning prompts, `llm_prediction` stores the raw structured response when the answer does not follow the short `Answer:` / `Reasoning:` format. This raw response should be parsed as JSON during reasoning evaluation.
